# Thesis documentation

This folder contains working materials for the master's thesis. The files here
are drafts and supporting assets; the final formatted document will be assembled
separately in Word/docx according to university requirements.

## Structure

- `drafts/00_keywords.md` — keywords, terminology, and abbreviations.
- `drafts/01_introduction.md` — introduction.
- `drafts/02_chapter_1_theory.md` — chapter I, theory.
- `drafts/03_chapter_2_design.md` — chapter II, system design.
- `drafts/04_chapter_3_implementation.md` — chapter III, implementation.
- `drafts/05_chapter_4_testing.md` — chapter IV, testing and results.
- `drafts/06_conclusion.md` — conclusion draft.
- `drafts/bibliography.md` — draft bibliography.
- `drafts/figures_plan.md` — planned diagrams, screenshots, and presentation visuals.
- `drafts/defense_questions.md` — possible defense questions and short answers.
- `figures/` — diagrams and architecture figures for the thesis.
- `screenshots/` — application screenshots for the thesis and presentation.
- `references/` — downloaded papers, reports, and source materials if redistribution is allowed.

## Markdown to Word workflow

1. Keep chapter text in `docs/drafts/*.md` while drafting and reviewing.
2. Copy each finished section into Word in thesis order: introduction, chapters I-IV, conclusion, bibliography.
3. Apply university formatting in Word: title styles, numbering, captions, page breaks, margins, and bibliography format.
4. Insert images from `figures/` and `screenshots/` manually so captions and cross-references can be checked.
5. Do a final pass in Word for page numbers, table of contents, figure numbering, and source formatting.

## Bibliography note

`drafts/bibliography.md` is a working bibliography. Every source must be
verified before final submission: author names, publication year, title,
publisher or venue, URL, access date, and formatting style.
