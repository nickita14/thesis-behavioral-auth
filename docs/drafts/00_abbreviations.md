# СПИСОК СОКРАЩЕНИЙ

AI — Artificial Intelligence; искусственный интеллект, область создания систем, способных решать задачи классификации, прогнозирования и принятия решений.

ML — Machine Learning; машинное обучение, подход к построению моделей на основе закономерностей в данных.

MFA — Multi-Factor Authentication; многофакторная аутентификация с использованием нескольких факторов проверки.

OTP — One-Time Password; одноразовый пароль для дополнительного подтверждения входа или операции.

API — Application Programming Interface; программный интерфейс взаимодействия между компонентами приложения.

REST — Representational State Transfer; архитектурный стиль построения web API на основе HTTP-ресурсов.

DRF — Django REST Framework; библиотека для разработки REST API в Django.

URL — Uniform Resource Locator; адрес web-ресурса.

HTML — HyperText Markup Language; язык разметки web-страниц.

JSON — JavaScript Object Notation; формат обмена структурированными данными.

HTTP — HyperText Transfer Protocol; протокол передачи данных в web-приложениях.

HTTPS — защищённая версия HTTP с использованием TLS/SSL.

TLS/SSL — криптографические протоколы защиты соединения между клиентом и сервером.

XGBoost — алгоритм градиентного бустинга над деревьями решений, используемый в phishing detection.

Isolation Forest — алгоритм обнаружения аномалий, основанный на изоляции нетипичных наблюдений в пространстве признаков.

Phishing — вид кибератаки и социальной инженерии, при котором злоумышленник имитирует доверенный ресурс для получения конфиденциальных данных.

Behavioral biometrics — поведенческая биометрия, метод проверки пользователя по устойчивым паттернам взаимодействия с системой.

Behavioral authentication — поведенческая аутентификация, подход к проверке пользователя на основе его действий и динамики поведения.

Keystroke dynamics — анализ временных характеристик набора текста: dwell time, flight time, скорость и ритм ввода.

Mouse dynamics — анализ поведения мыши или тачпада: движение курсора, клики, скроллинг, скорость и траектория.

Dwell time — время удержания клавиши между событием keydown и keyup.

Flight time — интервал между действиями над последовательными клавишами.

Feature extraction — преобразование исходных данных в числовые признаки для ML-модели.

Anomaly detection — обнаружение наблюдений, существенно отличающихся от нормального поведения.

Risk score — числовая оценка риска транзакции, URL или пользовательского действия.

Risk decision — итоговое решение системы: разрешить, запросить дополнительную проверку или отклонить операцию.

ALLOW — решение, при котором операция разрешается без дополнительной проверки.

CHALLENGE — решение, при котором требуется дополнительная проверка пользователя.

DENY — решение, при котором операция блокируется из-за высокого риска.

GDPR — General Data Protection Regulation; регламент ЕС по защите персональных данных.

SHA-256 — криптографическая хеш-функция, используемая для хранения хеша вместо исходного значения.

Audit log — журнал аудита проверок, транзакций и решений системы.

Model artifact — сохранённый файл обученной ML-модели, используемый приложением для inference.

Inference — применение обученной ML-модели к новым данным для получения прогноза или оценки риска.
