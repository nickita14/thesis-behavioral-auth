# КЛЮЧЕВЫЕ СЛОВА, ТЕРМИНОЛОГИЯ И СОКРАЩЕНИЯ

Ключевые слова: поведенческая биометрия, поведенческий анализ, искусственный интеллект, машинное обучение, аутентификация транзакций, phishing, фишинг, обнаружение аномалий, keystroke dynamics, mouse dynamics, risk scoring, XGBoost, Isolation Forest, Django, REST API, кибербезопасность, предотвращение мошенничества, онлайн-транзакции.

AI (Artificial Intelligence) — искусственный интеллект, область компьютерных наук, связанная с созданием систем, способных выполнять задачи, которые обычно требуют интеллектуального поведения человека, включая распознавание закономерностей, классификацию, прогнозирование и принятие решений.

ML (Machine Learning) — машинное обучение, направление искусственного интеллекта, основанное на построении моделей, которые выявляют закономерности в данных и используют их для прогнозирования или классификации новых наблюдений.

Behavioral biometrics — поведенческая биометрия, метод идентификации или проверки пользователя на основе устойчивых паттернов его взаимодействия с системой, таких как ритм набора текста, движения мыши, скорость реакции, особенности кликов и другие поведенческие признаки.

Behavioral authentication — поведенческая аутентификация, подход к проверке подлинности пользователя, при котором анализируются не только знания пользователя, например пароль, но и характер его поведения во время взаимодействия с приложением.

Continuous authentication — непрерывная аутентификация, механизм, при котором проверка пользователя выполняется не только в момент входа в систему, но и в процессе дальнейшей работы с приложением.

Keystroke dynamics — динамика нажатия клавиш, направление поведенческой биометрии, анализирующее временные характеристики ввода текста: длительность удержания клавиши, интервалы между нажатиями, скорость набора и устойчивость клавиатурного ритма.

Dwell time — время удержания клавиши, то есть интервал между событием нажатия клавиши и событием её отпускания.

Flight time — временной интервал между действиями над двумя последовательными клавишами, например между отпусканием одной клавиши и нажатием следующей.

Mouse dynamics — анализ поведения пользователя при работе с мышью или тачпадом, включающий траекторию движения курсора, скорость, ускорение, клики, скроллинг и паузы между действиями.

Phishing — вид социальной инженерии и кибератаки, при которой злоумышленник пытается получить конфиденциальные данные пользователя, например логин, пароль или платёжные реквизиты, путём имитации доверенного ресурса.

Phishing URL detection — обнаружение phishing-ссылок, задача классификации URL-адресов на легитимные и подозрительные или вредоносные с использованием лексических, хостовых, репутационных и содержательных признаков.

URL features — признаки URL-адреса, используемые для машинного анализа ссылки, например длина адреса, наличие IP-адреса вместо домена, количество поддоменов, наличие символа «@», использование HTTPS, возраст домена и другие характеристики.

Feature extraction — извлечение признаков, процесс преобразования исходных данных, таких как URL, HTML-код страницы или поведенческие события пользователя, в числовой набор характеристик, пригодный для обработки алгоритмами машинного обучения.

Anomaly detection — обнаружение аномалий, задача выявления наблюдений, существенно отличающихся от нормального поведения или ожидаемого распределения данных.

Isolation Forest — алгоритм машинного обучения для обнаружения аномалий, основанный на идее, что аномальные объекты легче изолировать в пространстве признаков, чем нормальные наблюдения.

XGBoost — библиотека и алгоритмический подход градиентного бустинга над деревьями решений, часто применяемый в задачах классификации, ранжирования и прогнозирования.

Risk score — числовая оценка риска, отражающая вероятность того, что действие пользователя, транзакция или URL-адрес являются подозрительными или потенциально вредоносными.

Risk decision — итоговое решение системы на основе оценки риска, например разрешить операцию, потребовать дополнительную проверку или заблокировать действие.

ALLOW — решение системы, означающее, что транзакция или действие пользователя признаны допустимыми без дополнительной проверки.

CHALLENGE — решение системы, означающее, что транзакция требует дополнительной проверки, например одноразового кода, подтверждения через email или ручной верификации.

DENY — решение системы, означающее блокировку транзакции или действия из-за высокого уровня риска.

MFA (Multi-Factor Authentication) — многофакторная аутентификация, метод проверки личности пользователя с использованием двух или более факторов: знания, владения и биометрических характеристик.

OTP (One-Time Password) — одноразовый пароль, используемый как дополнительный фактор подтверждения операции или входа в систему.

API (Application Programming Interface) — программный интерфейс приложения, определяющий правила взаимодействия между программными компонентами.

REST (Representational State Transfer) — архитектурный стиль построения web API, основанный на использовании HTTP-методов и ресурсов.

DRF (Django REST Framework) — библиотека для Django, предназначенная для разработки REST API.

Backend — серверная часть приложения, отвечающая за бизнес-логику, хранение данных, API, аутентификацию, интеграцию с ML-моделями и обработку запросов.

Frontend — клиентская часть приложения, с которой взаимодействует пользователь; в рамках данной работы frontend также отвечает за сбор поведенческих событий в браузере.

Django — web-фреймворк на языке Python, используемый для разработки backend-части приложения.

PostgreSQL — объектно-реляционная система управления базами данных, используемая в проекте для хранения пользователей, транзакций, поведенческих сессий, событий и результатов оценки риска.

JSON (JavaScript Object Notation) — текстовый формат обмена данными, используемый в REST API для передачи структурированной информации между frontend и backend.

JSONField — тип поля в Django/PostgreSQL, позволяющий хранить структурированные данные в формате JSON.

Cache — механизм временного хранения результатов вычислений или запросов, используемый для уменьшения повторной обработки дорогостоящих операций.

Audit log — журнал аудита, содержащий записи о выполненных проверках, транзакциях, решениях системы и связанных с ними параметрах.

Model artifact — сохранённый файл обученной модели машинного обучения, например в формате `.joblib`, используемый приложением для выполнения инференса.

Inference — процесс применения обученной ML-модели к новым данным для получения прогноза, вероятности или решения.

SOLID — набор принципов объектно-ориентированного проектирования, включающий Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation и Dependency Inversion.

SRP (Single Responsibility Principle) — принцип единственной ответственности, согласно которому класс или модуль должен иметь одну основную причину для изменения.

OCP (Open/Closed Principle) — принцип открытости/закрытости, согласно которому программные сущности должны быть открыты для расширения, но закрыты для изменения.

DIP (Dependency Inversion Principle) — принцип инверсии зависимостей, согласно которому высокоуровневые модули должны зависеть от абстракций, а не от конкретных реализаций.

GDPR (General Data Protection Regulation) — регламент Европейского союза по защите персональных данных, важный при проектировании систем, обрабатывающих пользовательские поведенческие данные.

SHA-256 — криптографическая хеш-функция, используемая для получения фиксированного хеша из входных данных; в рамках проекта применяется для исключения хранения исходных введённых символов.

UCI Phishing Websites Dataset — набор данных, содержащий признаки web-сайтов для обучения и оценки моделей обнаружения phishing-ресурсов.

CMU Keystroke Dynamics Benchmark Dataset — набор данных для исследования клавиатурной динамики, часто используемый в задачах поведенческой биометрии.
